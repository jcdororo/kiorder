# kiorder 수동 테스트 시나리오

> 테스트 환경: 백엔드(`localhost:3001`) + 프론트(`localhost:3000`) 로컬 실행 기준  
> 총 시나리오 수: 11개 흐름 / 체크리스트 항목 약 55개

---

## 사전 준비

- [ ] 백엔드 서버 실행 (`backend/` → `npm run start:dev`)
- [ ] 프론트 서버 실행 (`frontend/` → `npm run dev`)
- [ ] Supabase Realtime 활성화 확인 (DB → Publications → `Order`, `WaitingEntry` 체크)
- [ ] 테스트용 계정 확인 (STORE_OWNER / SYSTEM_ADMIN 각 1개)

---

## 시나리오 1 — 로그인 / 인증 / 라우트 가드

| # | 테스트 항목 | 기대 결과 |
|---|---|---|
| 1-1 | `localhost:3000` 접속 | 홈 페이지 (링크 목록) 표시 |
| 1-2 | `/login` 에서 STORE_OWNER 계정으로 로그인 | `/owner/dashboard` redirect |
| 1-3 | `/login` 에서 SYSTEM_ADMIN 계정으로 로그인 | `/system-admin/stores` redirect |
| 1-4 | 로그인 상태에서 `/login` 직접 접속 | role에 맞는 페이지로 redirect |
| 1-5 | 비로그인 상태에서 `/owner/dashboard` 직접 접속 | `/login` redirect |
| 1-6 | STORE_OWNER로 `/system-admin/stores` 접속 | `/unauthorized` redirect |
| 1-7 | 로그아웃 버튼 클릭 | 쿠키 삭제 후 `/login` redirect |
| 1-8 | 로그아웃 후 `/owner/dashboard` 직접 접속 | `/login` redirect |

## 시나리오 7 — 테이블오더 (`/table-order` → `/table-order/[tableId]/menu`)

| # | 테스트 항목 | 기대 결과 |
|---|---|---|
| 7-1 | `/table-order` 접속 | 테이블 목록(번호 그리드) 표시 |
| 7-2 | 테이블 선택 | `/table-order/[tableId]/menu` 이동 |
| 7-3 | 메뉴 페이지: 카테고리 클릭 | 해당 섹션으로 스크롤, 좌측 카테고리 하이라이트 |
| 7-4 | 스크롤 시 카테고리 자동 동기화 | 현재 섹션에 맞게 좌측 카테고리 하이라이트 변경 |
| 7-5 | 메뉴 + 버튼 클릭 | 우측 장바구니 패널에 항목 추가 |
| 7-6 | 장바구니 수량 증감 | 합계 금액 업데이트 |
| 7-7 | "주문" 버튼 클릭 | 주문 성공 모달 표시 (10초 카운트다운) |
| 7-8 | 모달 10초 후 자동 닫힘 | 장바구니 초기화, 계속 주문 가능 |
| 7-9 | 모달의 "주문 내역" 버튼 클릭 | 장바구니 패널 → 주문 내역 탭으로 전환 |
| 7-10 | "주문 내역" 탭 확인 | 메뉴명 × 수량 = 금액, 합계 표시 |
| 7-11 | "직원호출" 탭 | SERVICE 타입 메뉴만 표시, 수량 선택 후 주문 가능 |
